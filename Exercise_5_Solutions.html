Task 1

The idea behind using a pipeline is to avoid writing intermediate files to disk, reducing the need for I/O operations and speeding up the process. Each command takes its input from the previous one and passes its output to the next, using the pipe (|) symbol.

Pipeline Command

bwa mem -M tgmothYY.fa R1.fastq R2.fastq | \
samtools view -S -b - | \
samtools sort -o bwa_mem_out_sorted.bam && \
samtools index bwa_mem_out_sorted.bam
  • This command performs the alignment of the paired-end reads (R1.fastq and R2.fastq) against the reference genome (tgmothYY.fa).
  • Output: Instead of writing the output to a SAM file, it is passed directly to the next command through a pipe (|)
bwa mem -M tgmothYY.fa R1.fastq R2.fastq

samtools view:

  • This command converts the output from bwa mem (which is in SAM format) into BAM format.
  • The -S flag tells samtools that the input is in SAM format, and the -b flag specifies that the output should be in BAM format.
  • The - symbol tells samtools to read from standard input (which is the output of bwa mem).
  • Output: The BAM data is passed directly to the next command via another pipe.
samtools view -S -b -

samtools sort:

  • This command takes the unsorted BAM data from the previous step and sorts it by genomic coordinates.
  • The -o option specifies the output file, in this case, bwa_mem_out_sorted.bam.
  • Output: The sorted BAM file is written to bwa_mem_out_sorted.bam.
samtools sort -o bwa_mem_out_sorted.bam

samtools index:

  • This final command creates an index for the sorted BAM file, allowing efficient random access to specific regions of the file.
  • The && operator ensures that this command only runs if the previous commands succeeded.
samtools index bwa_mem_out_sorted.bam

Key Elements of the Pipeline:

  • Pipes (|): These are used to pass the output of one command directly as input to the next, without creating intermediate files. This reduces disk I/O and speeds up the workflow.
  • Redirection (-): The - symbol in samtools view tells it to read from standard input, which is the output of bwa mem.
  • Chaining with &&: The samtools index command is executed only if all previous steps succeed, ensuring that errors in alignment, conversion, or sorting stop the process before indexing

Bonus task 2: Use awk to find the chromosome with the highest coverage.

cat coverage_depth.txt | 
awk '{cov[$1] += $2} END {for (chr in cov) print chr, cov[chr]}' | \
sort -k2,2nr | head -n 1

awk '{cov[$1] += $2} END {for (chr in cov) print chr, cov[chr]}': The awk command aggregates the coverage depth ($2) by chromosome ($1). The associative array cov holds the sum of coverage depths for each chromosome. In the END block, it prints each chromosome and its total coverage.

sort -k2,2nr: This sorts the output by the second column (total coverage), in numerical and reverse order (nr), so the chromosome with the highest coverage appears first.

head -n 1: This extracts the first line, which corresponds to the chromosome with the highest total coverage.

Task 6: Print the first line of the alignment section of the bwa_mem_out.sam file. Identify the different fields. (Hint: Fields are separated by tab delimeters.

awk '$1 !~ /^@/ {print; exit}' bwa_mem_out.sam

$1 !~ /^@/: This checks that the first field ($1) does not start with an @ symbol, which indicates it's not a header line.

Header lines in a SAM file start with @ (i.e., @SQ, @PG), while alignment lines do not. {print; exit}: Once it finds the first line that matches the condition (i.e., the first alignment line), it prints that line and then exits, ensuring only the first alignment line is printed.
