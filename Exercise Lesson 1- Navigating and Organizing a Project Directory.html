Exercise Lesson 1: Navigating and Organizing a Project Directory

Tasks:

  1. Start by navigating to your home directory.

    • Verify your location using pwd.

  2. Create a directory for the project called MyProject

    • Confirm its creation by listing the contents of your current directory with ls

  3. Navigate into the MyProject directory.

    • Confirm your location using pwd

  4. Inside MyProject, create the following subdirectories:

    • src (for source code)
    • docs (for documentation)
    • tests (for test scripts)
    • List the contents of the current directory to confirm the creation of the subdirectories with ls

  5. Move up one level to your home directory (parent directory).

    • Verify your location using pwd.

  6. Navigate back into the MyProject/tests directory in one command.

    • Confirm your location using pwd

  7. In the tests directory, create a file called test_main.py

    • List the files in tests to confirm the creation of the file with ls.

  8. Navigate to the src directory using a relative path (go up one level, then into src)

  9. Inside the src directory, create a file called main.py.

    • List the files to confirm the creation of the file with ls

  10. Move the main.py file from src to the parent directory (MyProject)

    • Confirm that main.py has been moved by listing the contents of src and the parent directory with ls

  11. Navigate to the parent directory (MyProject) **and verify the location of main.py

    • List the contents to ensure main.py is now here with ls.

  12. Copy main.py into both src and tests directories using relative paths.

    • Verify the copy by listing the contents of both directories with ls.

  13. Now, move up to the parent directory (..) from your current location (inside MyProject) and list all project contents recursively to see the structure.

    • Use ls -R to check the structure.

  14. Navigate into the docs **directory and create a file called README.md

  15. Go back to the src **directory (navigate up one level, then into src)

  16. Remove the main.py file inside src

    • Confirm the deletion by listing the contents of src with ls.

  17. Move back to the MyProject directory (two levels up) and check the entire project structure recursively.

    • Use ls -R to verify the structure.

  18. Clean up by removing the entiretests directory.

    • List the contents of MyProject to verify that the tests directory has been removed with ls.

Solutions

  1. Navigate to your home directory: cd ~

  2. Create a directory for the project called MyProject: mkdir MyProject

  3. Navigate into the MyProject directory: cd MyProject

  4. Inside MyProject, create the following subdirectories: mkdir src docs tests

  5. Move up one level to your home directory (parent directory): cd ..

  6. Navigate back into the MyProject/tests directory in one command: cd MyProject/tests

  7. In the tests directory, create a file called test_main.py: touch test_main.py

  8. Navigate to the src directory using a relative path (go up one level, then into src): cd ../src

  9. Inside the src directory, create a file called main.py: touch main.py

  10. Move the main.py file from src to the parent directory (MyProject): mv main.py ..

  11. Navigate to the parent directory (MyProject) and verify the location of main.py: cd ..ls

  12. Copy main.py into both src and tests directories using relative paths: cp main.py src/cp main.py tests/

  13. Move up to the parent directory (..) and list all project contents recursively: cd ..ls -R MyProject

  14. Navigate into the docs directory and create a file called README.md: cd MyProject/docstouch README.md

  15. Go back to the src directory (navigate up one level, then into src): cd ../src

  16. Remove the main.py file inside src: rm main.py

  17. Move back to the MyProject directory (two levels up) and check the entire project structure recursively: cd ../..ls -R MyProject

  18. Clean up by removing the entire tests directory: rm -r MyProject/tests
