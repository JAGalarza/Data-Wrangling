Exercise 3. Grep and Regex

Lets use regular expressions to match different patterns from the file sample.txt file you created in Exercise 2.

This is the first line of the file.
The quick brown cat jumps over the lazy dog.
Linux v2.4 is a powerful operating system.
Linux V.3.6 allows you to build custom workflows.
Pipes are used to chain commands together.
Nano is a simple text editor.
Nano is often used for quick edits.
This is the last line of the file.

Tasks:

1. Basic Symbols

Write a regular expression that matches any word that starts with the letter "L" and ends with the letter "x".

Hint: Use the dot (.) symbol to match any character in between.

2. Character Classes

Write a regular expression to find all occurrences of the words "cat" or "dog".

Hint: Use a character class to match these words.

3. Quantifiers

Write a regular expression that matches any version numbers (e.g., v2.4, V.3.6) in the text.

Hint: Use the plus (+) or asterisk (*) to match multiple occurrences of digits and dots.

4. Grouping and Alternation

Write a regular expression to match lines that contain either the word "Linux" or "Nano".

Hint: Use parentheses (()) for grouping and the pipe (|) for alternation.

5. Escaped Characters

Write a regular expression that matches any line that ends with a period (.).

Hint: Remember to escape the dot character since it is a special character in regex.

6. Anchors

Write a regular expression that matches lines that start with "This".

Hint: Use the caret (^) symbol to anchor the match to the start of the line.

7. Special Characters

Write a regular expression that matches any word that contains the letter "o" exactly two times.

Hint: Combine character classes and quantifiers.

Bonus Tasks:

8. Word Boundaries

Write a regular expression to match the word "is" as a standalone word (not part of "This").

Hint: Use the \b word boundary anchor.

9. Case-Insensitive Search

Write a regular expression to match any occurrence of the word "nano" or "Nano", regardless of case.

Hint: Add a case-insensitive flag or use a character class to account for uppercase and lowercase letters.

Solutions

1.Basic Symbols. Find words that start with "L" and end with "x":

grep -o 'L.x' sample.txt
  • This regex matches any word starting with "L" and ending with "x", with any single character in between. The dot (.) represents any character except a newline.

2.Character Classes. Find occurrences of the words "cat" or "dog":

grep -E 'c[ao]t|dog' sample.txt
  • This regex matches either "cat" or "dog". The character class [ao] matches either "a" or "o". The pipe (|) alternates between "cat" and "dog".

3.Quantifiers. Find version numbers (e.g., v2.4, V.3.6):

grep -E '[vV]\d+\.\d+' sample.txt
  • This regex matches version numbers like v2.4 and V.3.6. The [0-9]+ matches one or more digits, and the escaped period (\.) matches a literal dot.

4.Grouping and Alternation. Find lines that contain either "Linux" or "Nano":

grep -E '(Linux|Nano)' sample.txt
  • This regex matches lines that contain either "Linux" or "Nano". Parentheses group the options, and the pipe (|) separates them for alternation.

5.Escaped Characters. Find lines that end with a period (.):

grep '\.$' sample.txt
  • This regex matches any line that ends with a period (.). Since the dot is a special character, it is escaped (\.) to match a literal period. The dollar sign ($) anchors the match to the end of the line.

6.Anchors. Find lines that start with "This":

grep '^This' sample.txt
  • This regex matches lines that start with "This". The caret (^) is used to anchor the match to the start of the line.

7.Special Characters. Find words that contain the letter "o" exactly twice:

grep -o '\b\w*o\w*o\w*\b' sample.txt
  • This regex matches words that contain the letter "o" exactly twice. \w* matches any number of word characters, and \b represents a word boundary.

8.Word Boundaries. Find the word "is" as a standalone word (not part of "This"):

grep -w 'is' sample.txt
  • This regex matches the word "is" as a standalone word.
    The \b anchors the match to word boundaries, ensuring "is" is not part of another word like "This".

9.Case-Insensitive Search. Find the word "nano" or "Nano" regardless of case:

grep -i 'nano' sample.txt
  • This regex matches the word "nano" or "Nano" regardless of case. The -i option makes the search case-insensitive.
